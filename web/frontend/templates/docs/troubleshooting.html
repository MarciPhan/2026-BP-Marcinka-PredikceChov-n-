Řešení potíží (Troubleshooting)

Systematický průvodce diagnostikou a opravou nejčastějších problémů s Metricord botem a dashboardem.

Rychlá diagnostika

Spusťte tyto příkazy jako první krok při jakémkoliv problému:

# 1. Je Redis funkční?
redis-cli ping
# Očekávaná odpověď: PONG

# 2. Žije bot?
redis-cli GET bot:heartbeat
# Odpověď: unix timestamp (pokud > 120s starý → bot spadl)

# 3. Na jaké servery je bot připojen?
redis-cli SCARD bot:guilds
# Odpověď: počet serverů

# 4. Kolik paměti Redis zabírá?
redis-cli INFO memory | grep used_memory_human

# 5. Běží procesy? (lokální)
ps aux | grep "bot/main.py"
ps aux | grep "uvicorn"

# 5b. Běží kontejnery? (Docker)
docker-compose ps

1. Problémy s botem

Bot se nespouští

Chybová hláška Příčina Řešení
LoginFailure Neplatný BOT_TOKEN Vygenerujte nový token v Discord Developer Portalu. Zkontrolujte .env (žádné uvozovky).
PrivilegedIntentsRequired Chybí Privileged Intents V Developer Portalu zapněte všechny 3 Privileged Intents (Presence, Members, Message Content).
Error 113: No route to host Redis nedostupný Spusťte Redis: redis-server --daemonize yes nebo docker-compose up -d redis.
[IDLE] Čekám na token BOT_TOKEN prázdný v .env Vyplňte BOT_TOKEN v .env. Bot se automaticky spustí do 30 sekund.

Bot nereaguje na příkazy

Postup diagnostiky:

  1. Zkontrolujte, zda je bot online na serveru (zelená tečka).
  2. Spusťte *sync (prefix příkaz) pro synchronizaci slash příkazů.
  3. Ověřte oprávnění bota na serveru — potřebuje View Channel, Send Messages, Use Slash Commands.
  4. Zkontrolujte logy: tail -20 bot.log
  5. Ověřte, zda příkaz není zakázán v config/config.py → COMMANDS_CONFIG.

Statistiky se neaktualizují

Možné příčiny a řešení:

  • Redis odpojení: redis-cli ping → pokud nepingne, restartujte Redis.
  • Bot v Rate Limit: Discord dočasně omezil bota. Počkejte 5–10 minut.
  • Kanál na blacklistu: Bot ignoruje kanály, ke kterým nemá přístup (View Channel).
  • Lite Mode: Dashboard bot (Lite Mode) netrackuje aktivitu — to dělá primární bot.

2. Problémy s dashboardem

Problém: „Unauthorized" nebo „Invalid Redirect URI"

Problém Řešení
Session vypršela Vymažte cookies pro doménu dashboardu. Zkuste Incognito režim.
Redirect URI nesedí V Discord Developer Portalu → OAuth2 přidejte přesnou URL: http://localhost:8092/auth/callback (nebo vaši produkční doménu).
Client Secret chybí Vyplňte DISCORD_CLIENT_SECRET v .env a restartujte.

Problém: Grafy jsou prázdné

  • Počkejte alespoň 15–30 minut od přidání bota na server.
  • Zkontrolujte, že bot má přístup ke kanálům (View Channel oprávnění).
  • Spusťte backfill pro historická data: /activity backfill days:30.
  • Ověřte, že Redis má správná data: redis-cli KEYS "stats:*:<guild_id>:*".

Problém: Port konflikty

# Zjistit, co zabírá port 8092
lsof -i :8092
# nebo
ss -tlnp | grep 8092

# Ukončit proces na portu
kill -9 $(lsof -t -i :8092)

3. Redis problémy

Redis Out of Memory

# Kontrola paměti
redis-cli INFO memory

# Řešení:
# 1. Zvýšit maxmemory v redis.conf
# 2. Vyčistit staré backfill klíče
redis-cli KEYS "backfill:*" | xargs redis-cli DEL

# 3. Kontrola velikosti sortovaných setů
redis-cli MEMORY USAGE "events:msg:<guild_id>:<user_id>"

Redis Persistence selhává

# Kontrola posledního uložení
redis-cli LASTSAVE

# Manuální save
redis-cli BGSAVE

# Zkontrolovat logy Redis
sudo journalctl -u redis -n 50

4. Docker problémy

Síťové problémy (botnet)

# Vytvořit síť pokud chybí
docker network create botnet

# Restart kontejnerů
docker-compose down
docker-compose up -d --build

# Kontrola logů
docker-compose logs -f --tail=50

5. Chybové kódy

Kód Význam Akce
ERR_REDIS_CONN Nelze se připojit k databázi. Zkontrolujte REDIS_URL v .env. Je Redis spuštěný?
ERR_DISCORD_403 Chybějící oprávnění (Forbidden). Pozvěte bota znovu s Administrator oprávněním.
ERR_DQS_LOW Nízká kvalita dat pro predikci. Sbírejte data déle (min. 7 dní) nebo spusťte backfill.
ERR_OAUTH_MISMATCH OAuth2 redirect URI nesouhlasí. Přidejte správné URI do Discord Developer Portalu.
ERR_TOKEN_INVALID Bot token je neplatný nebo expirovaný. Vygenerujte nový token v Developer Portalu.
Stále nefunguje? Zkontrolujte logy: tail -100 bot.log web.log. Výstup pošlete na Support Server nebo na support@metricord.app.
