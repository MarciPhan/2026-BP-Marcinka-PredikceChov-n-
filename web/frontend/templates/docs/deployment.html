Nasazení a údržba (Produkční průvodce)

Kompletní průvodce pro nasazení Metricord do produkce — od lokálního testu po zabezpečený server s SSL a monitoringem.

1. Systémové požadavky

Parametr Minimální Doporučeno (produkce)
OS Linux (libovolná distribuce) Ubuntu 22.04 LTS / Fedora 39+
CPU 1 jádro 2+ jádra (pro servery s 10k+ členy)
RAM 1 GB 2+ GB (Redis je in-memory)
Python 3.9 3.11+
Redis / Valkey 6.0 7.0+ (pro ACL podporu)
Docker (volitelné) 20.10 24.0+ s Compose v2

2. Příprava prostředí

2.1 Discord Developer Portal

Než začnete s nasazením, potřebujete:

  1. Jděte na Discord Developer Portal.
  2. Vytvořte novou aplikaci → sekce Bot → Reset Token → zkopírujte BOT_TOKEN.
  3. Zapněte všechny Privileged Gateway Intents:
    • Presence Intent — sledování online/offline statusu.
    • Server Members Intent — přístup k seznamu členů.
    • Message Content Intent — analýza zpráv (délka, ne obsah).
  4. V sekci OAuth2:
    • Přidejte Redirect URI: http://localhost:8092/auth/callback (pro vývoj).
    • Zkopírujte DISCORD_CLIENT_ID a DISCORD_CLIENT_SECRET.
  5. Pozvěte bota na server s oprávněním Administrator.

2.2 Generování secret klíčů

# Generování bezpečného secret klíče pro dashboard session
python3 -c "import secrets; print(secrets.token_hex(32))"

# Generování access tokenu pro API
python3 -c "import secrets; print(secrets.token_urlsafe(48))"

3. Metoda A: Lokální nasazení (start.sh)

Ideální pro vývoj a testování. Spustí všechny služby na jednom stroji.

1

Klonování a příprava

git clone https://github.com/MarciPhan/2026-BP-Marcinka-PredikceChov-n-.git
cd 2026-BP-Marcinka-PredikceChov-n-
cp .env.example .env
2

Konfigurace .env

Vyplňte všechny povinné proměnné:

# Povinné
BOT_TOKEN=your_discord_bot_token_here
DISCORD_CLIENT_SECRET=your_oauth2_client_secret

# Dashboard
DASHBOARD_PORT=8092
DASHBOARD_SECRET_KEY=<vygenerovaný hex klíč>
DASHBOARD_ACCESS_TOKEN=<vygenerovaný access token>

# Redis
REDIS_URL=redis://localhost:6379/0

# SMTP (volitelné — bez něj OTP kódy do konzole)
SMTP_PASSWORD=your_gmail_app_password
3

Spuštění

chmod +x start.sh
./start.sh

Skript automaticky:

  • ✅ Vytvoří Python virtuální prostředí (.venv)
  • ✅ Nainstaluje závislosti z requirements.txt
  • ✅ Spustí Redis/Valkey (pokud není aktivní)
  • ✅ Ukončí předchozí instance
  • ✅ Spustí bot (bot/main.py) a dashboard (uvicorn)

Dashboard bude dostupný na http://localhost:8092.

4. Metoda B: Docker Compose (doporučeno pro produkci)

1

Příprava sítě

# Docker síť "botnet" musí existovat
docker network create botnet 2>/dev/null || true
2

Konfigurace a spuštění

cp .env.example .env
nano .env    # vyplňte všechny proměnné

docker-compose up -d --build

Docker Compose spustí 4 služby:

Služba Container Účel
redis discord-redis In-memory databáze. Persistentní volume redis_data.
discord-bot-primary discord-bot-primary Hlavní bot — zpracovává příkazy, trackuje aktivitu.
discord-bot-dashboard discord-bot-dashboard Sekundární bot v Lite Mode — poskytuje guild data pro dashboard.
web-dashboard web-dashboard FastAPI dashboard na portu ${DASHBOARD_PORT}.

5. Produkční nasazení s Nginx a SSL

5.1 Nginx reverse proxy

Projektu je přiložena šablona v config/nginx-reverse-proxy.conf:

# /etc/nginx/sites-available/metricord
server {
    listen 80;
    server_name dashboard.example.com;

    location / {
        proxy_pass http://127.0.0.1:8092;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket support
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}

5.2 SSL s Let's Encrypt

# Instalace certbotu
sudo apt install certbot python3-certbot-nginx

# Získání certifikátu (automaticky upraví nginx config)
sudo certbot --nginx -d dashboard.example.com

# Automatická obnova (cron)
sudo certbot renew --dry-run
⚠️ Důležité Po nastavení SSL změňte Redirect URI v Discord Developer Portalu na https://dashboard.example.com/auth/callback a nastavte proměnnou DISCORD_REDIRECT_URI v .env.

5.3 Systemd služby (alternativa k Docker)

# /etc/systemd/system/metricord-bot.service
[Unit]
Description=Metricord Discord Bot
After=redis.service

[Service]
Type=simple
User=metricord
WorkingDirectory=/opt/metricord
Environment=PYTHONPATH=/opt/metricord
EnvironmentFile=/opt/metricord/.env
ExecStart=/opt/metricord/.venv/bin/python3 bot/main.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
# /etc/systemd/system/metricord-dashboard.service
[Unit]
Description=Metricord Dashboard
After=redis.service

[Service]
Type=simple
User=metricord
WorkingDirectory=/opt/metricord
Environment=PYTHONPATH=/opt/metricord
EnvironmentFile=/opt/metricord/.env
ExecStart=/opt/metricord/.venv/bin/python3 -m uvicorn web.backend.main:app --host 127.0.0.1 --port 8092
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
# Aktivace
sudo systemctl daemon-reload
sudo systemctl enable --now metricord-bot metricord-dashboard

6. Údržba a monitoring

6.1 Logy

Soubor Obsah Sledování
bot.log Discord eventy, chyby příkazů, Redis reconnecty tail -f bot.log
web.log HTTP requesty, OAuth sezení, chyby dashboardu tail -f web.log

Pro Docker: docker-compose logs -f

6.2 Health check

Bot zapisuje heartbeat do Redisu každých 60 sekund. Dashboard ho zobrazuje jako zelený/červený indikátor.

# Manuální kontrola heartbeatu
redis-cli GET bot:heartbeat
# Výsledek: timestamp posledního heartbeatu (pokud > 120s starý → bot je down)

6.3 Záloha Redis dat

# Manuální snapshot
redis-cli BGSAVE

# Kopírování zálohy (lokální Redis)
cp /var/lib/redis/dump.rdb /backup/metricord-$(date +%Y%m%d).rdb

# Docker volume záloha
docker cp discord-redis:/data/dump.rdb /backup/metricord-$(date +%Y%m%d).rdb

6.4 Aktualizace systému

# 1. Záloha dat
redis-cli BGSAVE

# 2. Pull nové verze
git pull origin main

# 3A. Lokální restart
./start.sh

# 3B. Docker restart
docker-compose up -d --build

7. Zastavení systému

# Lokální
pkill -f "bot/main.py"
pkill -f "uvicorn"

# Docker
docker-compose down

# Docker (s odstraněním volumes — SMAŽE DATA!)
docker-compose down -v

8. Production Checklist

  • ☐ BOT_TOKEN vyplněn a validní
  • ☐ DASHBOARD_SECRET_KEY vygenerován (32+ znaků)
  • ☐ DISCORD_CLIENT_SECRET vyplněn
  • ☐ Privileged Intents zapnuty (Presence, Members, Message Content)
  • ☐ OAuth2 Redirect URI odpovídá vaší doméně
  • ☐ Redis běží a je persistentní (RDB/AOF)
  • ☐ Redis port není vystaven na veřejnou síť
  • ☐ Nginx s SSL certifikátem (Let's Encrypt)
  • ☐ Firewall povoluje pouze porty 22, 80, 443
  • ☐ Záloha Redis dat nastavena (cron)
  • ☐ Logy rotovány (logrotate nebo Docker log driver)
  • ☐ Bot má na serveru roli s dostatečnými oprávněními
