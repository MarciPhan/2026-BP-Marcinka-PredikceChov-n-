Nasazení a údržba

Metricord je kontejnerizovaná aplikace. Doporučený způsob nasazení je pomocí Docker Compose.

1. Požadavky na systém

  • OS: Linux (Ubuntu 22.04+ doporučeno).
  • CPU: Minimálně 1 jádro (pro malé servery), 4+ jádra (pro velké komunity).
  • RAM: 2 GB+ (Redis potřebuje místo pro in-memory data).
  • Software: Docker 20.10+ a Docker Compose 2.0+.

2. Instalace (Krok za krokem)

1

Klonování repozitáře

git clone https://github.com/marcipan/metricord.git
cd metricord
2

Konfigurace prostředí

Zkopírujte vzorový soubor a vyplňte své údaje.

cp .env.example .env
nano .env

Nezapomeňte vyplnit BOT_TOKEN, DISCORD_CLIENT_ID a DISCORD_CLIENT_SECRET.

3

Spuštění kontejnerů

docker-compose up -d --build

Tento příkaz spustí Redis, Bot proces a Dashboard. Aplikace bude dostupná na portu 8093.

3. Údržba a debugování

Pro kontrolu stavu systému používejte následující příkazy:

  • Zobrazení logů: docker-compose logs -f
  • Restart systému: docker-compose restart
  • Čištění dat: docker exec -it metricord-redis redis-cli FLUSHALL (Varování: Smaže všechna data!).

4. Zálohování (Redis Persistence)

Metricord využívá Redis RDB snapshots každých 60 sekund a AOF logging pro maximální bezpečnost dat. Zálohy naleznete v adresáři ./redis_data/. Doporučujeme tento adresář pravidelně zálohovat mimo server.

⚠️ Aktualizace Před každou aktualizací (git pull) doporučujeme vytvořit snapshot Redisu pomocí příkazu redis-cli SAVE.
