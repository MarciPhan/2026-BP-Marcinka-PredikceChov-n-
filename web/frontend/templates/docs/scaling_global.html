Global Scaling & High Availability

Jak provozovat Metricord pro miliony uživatelů napříč kontinenty.

1. Redis Sentinel (HA)

Pro zajištění běhu i při výpadku hlavního databázového uzlu využijte Redis Sentinel. Ten automaticky zvolí nového Mastera, pokud hlavní uzel selže.

# Příklad sentinel.conf
sentinel monitor mymaster 127.0.0.1 6379 2
sentinel down-after-milliseconds mymaster 5000
sentinel failover-timeout mymaster 60000

2. Redis Sharding (Cluster)

Pokud se data (events) nevejdou do RAM jednoho serveru, využijte Redis Cluster pro horizontální škálování. Data budou rozdělena do 16,384 slotů napříč uzly.

3. Geo-Distributed Monitoring

Pro servery s globální komunitou doporučujeme nasadit "Edge" boty v různých regionech (US-East, EU-Central, ASIA-South), kteří budou data posílat do centrálního Redis clusteru.

Architektonický tip: Pro globální škálování je klíčová latence k Redisu. Vždy se snažte mít bota i Redis v rámci stejného regionu a stejné VPC.
