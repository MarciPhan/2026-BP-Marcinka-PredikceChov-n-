Security Hardening (Zabezpečení infrastruktury)

Metricord je bezpečný na aplikační úrovni, ale produkční server vyžaduje dodatečné zabezpečení na úrovni OS, sítě a kontejnerů. Tento průvodce pokrývá všechny vrstvy.

1. Firewall (UFW / firewalld)

Povolte pouze nezbytné porty. Redis nesmí být nikdy přístupný zvenčí.

# Ubuntu / Debian (UFW)
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP (redirect na HTTPS)
sudo ufw allow 443/tcp   # HTTPS
sudo ufw enable

# Fedora / RHEL (firewalld)
sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
⚠️ Kritické Port 6379 (Redis) nesmí být přístupný z internetu. Pokud používáte Docker, Redis komunika pouze přes interní síť botnet.

2. Redis zabezpečení

2.1 Heslo (requirepass)

# V redis.conf
requirepass your_strong_password_here_32chars

# V .env
REDIS_URL=redis://:your_strong_password_here_32chars@localhost:6379/0

2.2 ACL (Redis 6.0+)

Vytvořte oddělené účty pro bota a dashboard s minimálními oprávněními:

# redis.conf nebo redis-cli ACL SETUSER
user bot on >bot_password ~events:* ~stats:* ~hll:* ~presence:* ~bot:* ~user:* +@all -@admin
user dashboard on >dash_password ~stats:* ~hll:* ~presence:* ~bot:heartbeat +get +hget +hgetall +zrange +pfcount +scard +smembers
user default off

2.3 Redis TLS (Redis 6.0+)

# V redis.conf
tls-port 6379
port 0
tls-cert-file /path/to/redis.crt
tls-key-file /path/to/redis.key
tls-ca-cert-file /path/to/ca.crt

# V .env
REDIS_URL=rediss://:password@localhost:6379/0  # rediss:// = TLS

3. Docker zabezpečení

  • Non-root user: Dockerfile používá python:3.9-slim. Přidejte: 
    RUN useradd -m metricord
USER metricord
  • Read-only filesystem: 
    # docker-compose.yml
services:
  web-dashboard:
    read_only: true
    tmpfs:
      - /tmp
  • Resource limits: 
    services:
  discord-bot-primary:
    deploy:
      resources:
        limits:
          memory: 512M
          cpus: '1.0'
  • Network isolation: Všechny služby komunikují přes privátní síť botnet. Pouze dashboard vystavuje port.

4. SSH zabezpečení

# /etc/ssh/sshd_config
PermitRootLogin no
PasswordAuthentication no       # Používejte SSH klíče
MaxAuthTries 3
AllowUsers metricord            # Povolte jen vašeho uživatele

5. Fail2Ban

Ochrana proti bruteforce útokům na SSH a dashboard:

sudo apt install fail2ban    # Ubuntu/Debian
sudo dnf install fail2ban    # Fedora

# Konfigurace: /etc/fail2ban/jail.local
[sshd]
enabled = true
port = ssh
maxretry = 3
bantime = 3600

[nginx-http-auth]
enabled = true
port = http,https
maxretry = 5
bantime = 600

6. Bezpečnostní checklist

  • ☐ Redis port 6379 není přístupný z internetu
  • ☐ Redis má nastaveno requirepass s 32+ znakovým heslem
  • ☐ .env soubor má oprávnění 600 (chmod 600 .env)
  • ☐ SSH nepovoluje root login ani password auth
  • ☐ Firewall povoluje pouze porty 22, 80, 443
  • ☐ Fail2Ban aktivní pro SSH a Nginx
  • ☐ Docker kontejnery nemají --privileged
  • ☐ SSL certifikát na dashboardu (Let's Encrypt)
  • ☐ DASHBOARD_SECRET_KEY je unikátní a silný
  • ☐ Automatické zálohy Redis dat mimo server
💡 Audit Pravidelně kontrolujte přístupy přes redis-cli ACL LIST a prohlížejte logy Fail2Ban: sudo fail2ban-client status sshd.
