Security Hardening (Zabezpečení serveru)

Metricord je bezpečný, ale váš server musí být také. Tento průvodce ukazuje, jak ochránit systém na úrovni OS.

1. Firewall (UFW)

Povolte pouze nezbytné porty. Redis by neměl být nikdy přístupný zvenčí.

ufw default deny incoming
ufw allow 22/tcp # SSH
ufw allow 80/tcp # HTTP
ufw allow 443/tcp # HTTPS
ufw enable

2. Redis ACL (Access Control Lists)

Pokud používáte Redis 6.0+, zaveďte uživatelské účty s omezenými právy pro bota a dashboard.

user bot on >password ~events:* ~stats:* +@all -@admin
user dashboard on >password ~stats:* +get +set

3. Fail2Ban

Nainstalujte Fail2Ban pro ochranu SSH a dashboardu (pokud zaznamenáte bruteforce útoky na login).

apt install fail2ban
# Konfigurace v /etc/fail2ban/jail.local
⚠️ Pozor: Nikdy nepoužívejte výchozí hesla pro Redis nebo Docker API. Vždy generujte silné 32-znakové řetězce do .env.
