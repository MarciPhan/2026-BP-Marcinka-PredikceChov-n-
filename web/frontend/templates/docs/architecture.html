Architektura systému

Metricord je navržen jako distribuovaný systém s důrazem na extrémní výkon a nízkou latenci při zpracování milionů událostí.

1. Technický stack

  • Backend (Bot): Python 3.10+, discord.py (Asynchronní zpracování událostí).
  • Backend (Dashboard): FastAPI (High-performance web server).
  • Datové úložiště: Redis (In-memory databáze pro real-time analytiku).
  • Analytika: NumPy/SciPy pro matematické modely, Chart.js pro vizualizaci.

2. Tok dat (Data Flow)

Zde je zjednodušený pohled na to, co se děje s vaší zprávou:

  1. Event Capture: Bot zachytí událost on_message přes Discord Gateway.
  2. Pre-processing: Vypočítá se váha zprávy a XP (lokálně v RAM bota).
  3. Redis Push: Data se atomicky zapíší do Redisu (využíváme Sorted Sets pro časové řady a HyperLogLog pro unikátní uživatele).
  4. Dashboard Read: Při návštěvě webu FastAPI přečte agregovaná data z Redisu a pošle je do prohlížeče.

3. Proč Redis?

Na rozdíl od tradičních SQL databází nám Redis umožňuje:

  • Zpracovat tisíce zpráv za sekundu s latencí pod 1ms.
  • Ukládat miliardy unikátních interakcí s minimální spotřebou paměti díky Probabilistic Data Structures.
  • Jednoduchou horizontální škálovatelnost.

4. Bezpečnostní vrstvy

Systém používá víceúrovňovou ochranu:

  • Discord OAuth2: Nikdy nevidíme vaše heslo k Discordu.
  • JWT Tokens: Sezení v dashboardu je zabezpečeno kryptograficky podepsanými tokeny.
  • Rate Limiting: Ochrana proti DDoS útokům na úrovni API.
Důležité pro vývojáře: Metricord je navržen tak, aby bot mohl běžet na jiném serveru než dashboard. Jediným pojítkem je přístup k identické instanci Redis.
