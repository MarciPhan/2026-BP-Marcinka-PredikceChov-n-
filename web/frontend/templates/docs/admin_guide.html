Administrátorský průvodce (Instance Manager)

Tento průvodce je určen pro ty, kteří spravují samotnou instanci Metricord (backend, databáze, konfigurace).

1. Škálování (Scaling Limits)

Výchozí instalace na jednom VPS pohodlně zvládne až 50,000 členů. Pro větší servery (100k+) doporučujeme:

  • Redis Slaves: Rozdělení čtení (Dashboard) a zápisu (Bot) na různé instance Redisu.
  • Event Thinning: Snížení frekvence sběru voice sezení z 1 minuty na 5 minut v activity.py.

2. Performance Tuning (Redis)

Parametr Doporučená hodnota Dopad
tcp-keepalive 60 Stabilnější bot připojení
maxmemory 75% system gold Prevence swapování na disk

3. Správa vah (XP Tuning)

Pravidla pro výpočet XP můžete měnit v reálném čase bez restartu bota. To se děje přes Redis dashboard nebo přes API endpoint /admin/config/weights.

  • Výchozí stav: 1 char = 1 XP, Reply = 60 XP.
  • Doporučení: Pokud máte příliš rychlý nárůst úrovní, zvyšte váhu pro bans nebo zaveďte přísnější timeouty sezení.

2. Údržba Redisu

Redis uchovává historická data v operační paměti. Pro servery s 1M+ zprávami měsíčně doporučujeme:

  • Nastavit maxmemory-policy na noeviction (aby nedocházelo k mazání historických statistik).
  • Pravidelně provádět BGSAVE pro perzistenci na disk.
  • Sledovat velikost Sorted Setů (ZSET) u klíčů events:*.

3. Správa bot tokenů

Při změně tokenu bota v .env je nutné provést kompletní restart kontejnerů:

docker-compose down
docker-compose up -d

4. Logování a Monitoring

Metricord generuje dva hlavní logy:

  • bot.log: Události z Discordu, chyby v příkazech, Redis reconnecty.
  • web.log: HTTP požadavky, OAuth sezení, chyby v renderování dashboardu.
Důležité upozornění: Nikdy nesdílejte .env soubor ani SECRET_KEY. Kompromitace těchto klíčů umožňuje komukoliv ovládat váš dashboard a přistupovat k datům všech uživatelů.
