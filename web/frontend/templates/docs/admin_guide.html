Administrátorský průvodce (Instance Manager)

Průvodce pro správce, kteří provozují vlastní instanci Metricord — konfigurace, škálování, Redis údržba, monitoring a optimalizace výkonu.

1. Konfigurace prostředí (.env)

Kompletní přehled všech environmentálních proměnných:

Proměnná Povinná Výchozí Popis
BOT_TOKEN ✅ — Discord bot token z Developer Portalu.
DISCORD_CLIENT_SECRET ✅ — OAuth2 Client Secret pro dashboard přihlášení.
DASHBOARD_PORT — 8092 Port pro web dashboard.
DASHBOARD_SECRET_KEY ✅ — Kryptografický klíč pro podpis session cookies. Generujte: python3 -c "import secrets; print(secrets.token_hex(32))"
DASHBOARD_ACCESS_TOKEN — — Bearer token pro API přístup (volitelný).
REDIS_URL — redis://localhost:6379/0 Redis connection string. Podpora hesla: redis://:password@host:6379/0
SMTP_PASSWORD — — Heslo pro SMTP (e-mailový OTP). Bez nastavení = OTP do konzole.
BOT_LITE_MODE — 0 Nastavte na 1 pro sekundární bot instanci (dashboard bot).
DISCORD_REDIRECT_URI — http://localhost:{PORT}/auth/callback OAuth2 redirect URI. Nastavte pro produkci s vlastní doménou.

2. Škálování (Scaling Limits)

Velikost serveru RAM Redis CPU Poznámky
< 5,000 členů 256 MB 1 jádro Výchozí konfigurace bez úprav.
5k – 50k členů 1 GB 2 jádra Zvažte zvýšení maxmemory v Redis config.
50k – 100k+ členů 4+ GB 4+ jader Redis read replicas, event thinning (voice sezení z 1 min na 5 min).

3. Redis Performance Tuning

Parametr Doporučená hodnota Efekt
tcp-keepalive 60 Stabilnější bot↔Redis připojení, prevence TCP timeout.
maxmemory 75% system RAM Prevence swapování na disk — fatální pro in-memory DB.
maxmemory-policy noeviction Nikdy automaticky nemazat data. Lepší error než ztráta statistik.
save 300 10 RDB snapshot každých 5 minut pokud bylo 10+ změn.
appendonly yes AOF logging pro maximální bezpečnost dat. 
# Příklad redis.conf optimalizace
tcp-keepalive 60
maxmemory 1gb
maxmemory-policy noeviction
save 300 10
appendonly yes
appendfsync everysec

4. Správa vah XP (bez restartu)

XP pravidla lze měnit v reálném čase přes Redis nebo API:

# Přes redis-cli
redis-cli HSET config:xp:weights msg_short 1 msg_medium 5 msg_long 15 reply_bonus 10 voice_per_min 5

# Přes API (pokud máte access token)
curl -X POST http://localhost:8092/admin/config/weights \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"msg_short": 1, "msg_medium": 5, "msg_long": 15}'

5. Správa bot tokenů

Při změně tokenu:

# Lokální
pkill -f "bot/main.py"
# Upravte .env
./start.sh

# Docker
docker-compose down
# Upravte .env
docker-compose up -d
⚠️ Bot Lite Mode Docker Compose provozuje dvě bot instance — primární (plný tracking) a dashboard bot (Lite Mode). Lite Mode přeskakuje interaktivní příkazy a slouží pouze pro poskytování guild dat dashboardu. Pokud nepotřebujete dvě instance, zakomentujte službu discord-bot-dashboard.

6. Logování a Monitoring

Log Obsah Kde
bot.log Discord eventy, chyby příkazů, Redis reconnecty, guild join/leave. Kořen projektu / Docker stdout
web.log HTTP requesty, OAuth sezení, API chyby, rendering exceptions. Kořen projektu / Docker stdout
Discord Console Channel Startup diagnostika, modul loading, sync status. Kanál ID v config/config.py

Monitorovací příkazy

# Redis info
redis-cli INFO memory | grep used_memory_human
redis-cli INFO keyspace
redis-cli DBSIZE

# Bot heartbeat
redis-cli GET bot:heartbeat

# Počet tracked guilds
redis-cli SCARD bot:guilds

# Online členové na guild
redis-cli GET presence:online:<guild_id>
🔒 Bezpečnostní upozornění Nikdy nesdílejte .env soubor ani SECRET_KEY. Kompromitace těchto klíčů umožňuje komukoliv ovládat váš dashboard a přistupovat k datům všech uživatelů.
