Developer Guide: Vývoj a rozšiřování Metricord

Průvodce pro vývojáře, kteří chtějí nasadit, upravovat nebo rozšiřovat Metricord. Pokrývá lokální vývojové prostředí, strukturu modulů, coding conventions a testování.

1. Lokální vývojové prostředí

# Klonování
git clone https://github.com/MarciPhan/2026-BP-Marcinka-PredikceChov-n-.git
cd 2026-BP-Marcinka-PredikceChov-n-

# Virtuální prostředí
python3 -m venv .venv
source .venv/bin/activate

# Závislosti
pip install -r requirements.txt

# Konfigurace
cp .env.example .env
# Vyplňte alespoň BOT_TOKEN a REDIS_URL

# Redis (musí běžet)
redis-server --daemonize yes   # nebo: sudo systemctl start redis

# Spuštění bota
export PYTHONPATH=$PWD
python3 bot/main.py

# Spuštění dashboardu (v novém terminálu)
export PYTHONPATH=$PWD
python3 -m uvicorn web.backend.main:app --host 0.0.0.0 --port 8092 --reload
Uvicorn --reload Flag --reload automaticky restartuje dashboard při změně souborů. Ideální pro vývoj frontend šablon.

Shared modul

Sdílený kód mezi botem a dashboardem:

  • shared/keys.py — Centrální definice Redis klíčů. Všechny klíče na jednom místě.
  • shared/models.py — Matematické modely (Markov, Kaplan-Meier). Třída CommunityModels.
  • shared/redis_client.py — Singleton async Redis klient.

3. Přidání nového příkazu (Cog)

# bot/commands/my_command.py
import discord
from discord.ext import commands
from shared.redis_client import get_redis_client

class MyCommand(commands.Cog):
    def __init__(self, bot):
        self.bot = bot

    @discord.app_commands.command(name="mycommand", description="Popis příkazu")
    async def my_command(self, interaction: discord.Interaction):
        r = await get_redis_client()
        # ... vaše logika ...
        await r.close()
        await interaction.response.send_message("Hotovo!", ephemeral=True)

async def setup(bot):
    await bot.add_cog(MyCommand(bot))

Soubor bude automaticky načten při dalším restartu bota (díky auto-discovery v load_commands()).

4. Přidání nové metriky do dashboardu

  1. Definujte Redis klíč v shared/keys.py: 
    def K_CUSTOM(gid: int) -> str:
    return f"stats:custom:{gid}"
  2. Zapisujte data v bot commandu nebo listeneru: 
    r = await get_redis_client()
await r.hincrby(K_CUSTOM(guild.id), "field", 1)
  3. Přidejte API endpoint v web/backend/main.py: 
    @app.get("/api/guild/{guild_id}/custom")
async def get_custom(guild_id: int):
    r = await get_redis_client()
    data = await r.hgetall(f"stats:custom:{guild_id}")
    await r.close()
    return data
  4. Vytvořte widget v dashboard šabloně pomocí Chart.js.

5. Přidání vlastního prediktivního modelu

Predikční engine v web/backend/utils.py je modulární. Přidejte novou async funkci:

# V utils.py nebo jako nový modul
async def predict_participation(r, guild_id: int) -> dict:
    """Predikce účasti na základě historických vzorců."""
    # Čtení dat z Redisu
    events = await r.zrangebyscore(
        f"events:msg:{guild_id}:*", "-inf", "+inf"
    )
    # Vaše logika (numpy, sklearn, ...)
    model_output = {"prediction": 0.85, "confidence": 0.72}
    return model_output

6. Coding Conventions

  • Async everywhere: Vždy používejte async/await pro IO operace. Bot zpracovává stovky eventů za sekundu.
  • Redis přes shared client: Používejte get_redis_client() z shared/redis_client.py. Nezakládejte vlastní connection pools.
  • Klíče přes keys.py: Nikdy nepoužívejte hardcoded Redis klíče. Vždy definujte v shared/keys.py.
  • .env pro secrets: Žádné tokeny nebo hesla v kódu. Všechny secrets musí být v .env.
  • Ephemeral responses: Pro příkazy s citlivými informacemi používejte ephemeral=True.
  • Error handling: V Cog příkazech vždy ošetřete chyby. Uživatel nesmí vidět Python traceback.

Testy jsou v kořenovém adresáři projektu:

Test Účel Spuštění

Spuštění testů vyžaduje běžící Redis Ujistěte se, že Redis je dostupný na localhost:6379 předtím, než spustíte testy. Testy zapisují do databáze 0 (výchozí), ale používají prefixované klíče.
