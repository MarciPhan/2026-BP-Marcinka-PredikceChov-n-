Redis Datové schéma (Kompletní referenční mapa)

Metricord využívá Redis jako primární databázi pro in-memory analytiku. Tato stránka dokumentuje všechny klíče definované v shared/keys.py a runtime klíče vytvářené botem.

💡 Konvence pojmenování klíčů Formát: {kategorie}:{subtyp}:{guild_id}:{identifikátor}. Všechny klíče jsou definovány centrálně v shared/keys.py. Nikdy nepoužívejte hardcoded klíče v kódu.

1. Analytické eventy (Time-Series)

Data jednotlivých uživatelů jako Sorted Sets (score = UNIX timestamp).

Klíč (Pattern) Typ TTL Popis Funkce v keys.py
events:msg:{gid}:{uid} Sorted Set — Timestampy zpráv uživatele. Member = JSON metadata (délka, kanál, reply). K_EVENTS_MSG()
events:voice:{gid}:{uid} Sorted Set — Voice sezení — start/end časy, kanál, délka trvání. K_EVENTS_VOICE()
events:action:{gid}:{uid} Sorted Set — Moderátorské akce (ban, kick, timeout, delete) s timestampem. K_EVENTS_ACTION()

2. Agregované statistiky

Klíč (Pattern) Typ TTL Popis Funkce v keys.py
hll:dau:{gid}:{YYYYMMDD} HyperLogLog — Unikátní aktivní uživatelé za den. ~12 KB / den / guild. Chyba: 0.81%. K_DAU()
stats:hourly:{gid}:{YYYYMMDD} Hash — Počet zpráv za každou hodinu (field = "0"–"23", value = count). K_HOURLY()
stats:heatmap:{gid} Hash — Aktivita po dnech a hodinách (field = "0:14" = pondělí 14:00). K_HEATMAP()
stats:msglen:{gid} Hash — Distribuce délky zpráv (field = bucket "0-15", "15-100", "100+"). K_MSGLEN()
stats:total_msgs:{gid} String — Kumulativní celkový počet zpráv na serveru. K_TOTAL_MSGS()

3. Uživatelská data

Klíč (Pattern) Typ TTL Popis Funkce v keys.py
user:info:{uid} Hash 7 dní Cached uživatelské údaje: name, avatar, roles. K_USER_INFO()

4. Konfigurace a systémové klíče

Klíč (Pattern) Typ TTL Popis Funkce v keys.py
cfg:logchan:{gid} String — ID log kanálu pro daný server. K_LOGCHAN()
backfill:progress:{gid} String — Status průběhu backfillu (procenta, status). K_BACKFILL_PROGRESS()

5. Runtime klíče (bot/main.py)

Tyto klíče nejsou v keys.py, ale zapisuje je bot přímo:

Klíč (Pattern) Typ TTL Popis
bot:heartbeat String — UNIX timestamp posledního heartbeatu (60s interval). Dashboard ho čte pro UP/DOWN status.
bot:guilds Set — ID všech serverů, kde je bot přítomen.
bot:guilds:primary Set — Servery primárního bota.
bot:guilds:dashboard Set — Servery dashboard bota (Lite Mode).
bot:token String — Volitelný token uložený přes dashboard (když .env je prázdný).
presence:online:{gid} String 60s Aktuálně online členové. Zapisuje member_stats_task každých 10s.
presence:total:{gid} String 60s Celkový počet členů serveru.
guild:verification_level:{gid} String — Úroveň verifikace serveru (pro Security Score).
guild:mfa_level:{gid} String — MFA požadavek pro moderátory.
guild:explicit_filter:{gid} String — Explicit content filter nastavení.

6. Discourse klíče

Klíč (Pattern) Typ Popis Funkce v keys.py
discourse:conf:{gid} Hash Konfigurace Discourse fóra pro guild. K_DISCOURSE_CONF()
user:discourse:{uid} Set Seznam Discourse guild IDs vlastněných uživatelem. K_USER_DISCOURSE()
discourse:ids Set Všechna registrovaná Discourse guild IDs. K_DISCOURSE_IDS()

7. Diagnostické příkazy

# Zobrazit všechny klíče pro guild
redis-cli SCAN 0 MATCH "*:{guild_id}*" COUNT 100

# Počet členů v DAU HLL
redis-cli PFCOUNT "hll:dau:{guild_id}:{yyyymmdd}"

# Počet zpráv uživatele
redis-cli ZCARD "events:msg:{guild_id}:{user_id}"

# Paměťová náročnost klíče
redis-cli MEMORY USAGE "events:msg:{guild_id}:{user_id}"

# Celková velikost databáze
redis-cli DBSIZE
⚠️ Výkon Při přímém přístupu k Redisu doporučujeme SCAN místo KEYS, aby nedošlo k blokování databáze při velkém objemu dat. Příkaz KEYS je O(N) a může na velkých databázích trvat sekundy.
