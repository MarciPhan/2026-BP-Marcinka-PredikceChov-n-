Vnitřní datové schéma (Redis Map)

Metricord využívá Redis jako primární databázi pro in-memory analytiku. Zde je kompletní mapa klíčů a datových typů používaných v systému.

1. Uživatelská data a stavy

Klíč (Pattern) Typ Expirace Popis
user:info:{uid} Hash 7 dní Ukládá name, avatar a roles (IDs).
activity:state:{gid}:{uid}:{key} String - Dočasné značky pro start/konec sezení (chat_start, voice_start).

2. Analytické eventy (Time-Series)

Pro každou metriku používáme Sorted Sets (ZSET), kde skóre je UNIX timestamp a hodnota je JSON s detaily eventu.

  • events:msg:{gid}:{uid}: Historie zpráv (jen metadata: délka, reply).
  • events:voice:{gid}:{uid}: Historie voice sezení (duration, channel).
  • events:action:{gid}:{uid}: Moderační akce (type: kick/ban/timeout).

3. Agregované statistiky (HyperLogLog)

Používáme pro extrémně rychlý výpočet unikátních uživatelů (DAU/WAU/MAU).

  • stats:hll:{gid}:day:{YYYY-MM-DD}: Unikátní uživatelé za den.
  • stats:hll:{gid}:week:{YYYY-WW}: Unikátní za týden.
  • stats:hll:{gid}:month:{YYYY-MM}: Unikátní za měsíc.

4. Systémová konfigurace

  • bot:guilds: Set všech ID serverů, kde je bot přítomen.
  • config:action_weights: Hash uživatelsky definovaných vah pro XP (bans=300, msg=1 atd.).
💡 Tip pro vývojáře: Při přímém přístupu k Redisu doporučujeme používat příkaz SCAN místo KEYS, aby nedošlo k blokování databáze při velkém objemu dat.
